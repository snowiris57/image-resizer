# 📦 画像リサイズくん

LINEで画像を送るとき、20MBを超えると「写真」ではなく「ファイル」として扱われてしまう問題を解決するツールです。
JPG・PNG・BMP を選ぶだけで、20MB以下に自動で圧縮します。

**👉 使う: https://snowiris57.github.io/image-resizer/**

## 使い方

1. 上のURLをブラウザで開く
   (または `index.html` をPCに保存してダブルクリックでも動きます。オフラインOK)
2. 画像をドラッグ&ドロップ、またはタップして選択(複数まとめてOK)
   - **フォルダごとの一括変換もOK**: フォルダをドラッグ&ドロップするか「📁 フォルダごと選択」ボタンから。サブフォルダの中身も全部処理されます
3. 自動で20MB以下に圧縮されるので「保存」ボタンでダウンロード
4. ダウンロードした画像をLINEで送る

## 仕様

- 対応形式: JPG / PNG / BMP(入力)
- 上限サイズは 20MB(LINE用・既定)/ 10MB / 5MB / 3MB から選択可
- 20MB以下のJPG/PNGは圧縮せず「そのまま送れます」と表示
- BMPはサイズに関わらずJPGに変換(LINEはBMPをファイル扱いするため)
- 透過PNGは透過を保ったままPNGで縮小、それ以外はJPGに変換
- 「EXIF(撮影情報)を引き継ぐ」チェック(既定ON)で、JPG→JPG変換時に撮影日時・カメラ情報・PPIなどのEXIFとICCカラープロファイルを引き継ぐ(回転タグは二重回転しないよう補正)。OFFでメタデータ除去
- 上限(の約95%)に収まる範囲で最高の画質を二分探索で選ぶ。圧縮しすぎず、できるだけ高画質を保つ
- フォルダ一括変換に対応(サブフォルダも再帰的に処理、画像以外のファイルは自動スキップ)
- 複数枚はZIPでまとめてダウンロード可(ZIP内は元のフォルダ構成を維持)
- ZIP保存なら元ファイルの更新日時も引き継ぎ(解凍時に復元される)。単体の「保存」ボタンはブラウザの仕様上ダウンロード時刻になる
- メモリ不足を防ぐため同時に処理するのは2枚まで(残りは順番待ちで自動処理)
- 画像は一切サーバーに送信されず、すべてブラウザ内で処理

## 技術メモ

- 依存ライブラリなしの単一HTMLファイル(Canvas APIで圧縮、ZIPも自前実装)
- Squoosh / browser-image-compression と同じクライアントサイド圧縮方式
- EXIF回転は `createImageBitmap` の `imageOrientation: "from-image"` で補正
- `main` ブランチへのpushでGitHub Actionsが自動的にGitHub Pagesへデプロイ

# 📸 連続スクショくん

Web会議の画面に映っているPDFや資料を、ページ送りに合わせて自動でスクショ保存するツールです。

**👉 使う: https://snowiris57.github.io/image-resizer/capture/**

## 使い方

1. 上のURLをブラウザで開き、「キャプチャ開始」を押す
2. ブラウザのダイアログで会議アプリのウィンドウ(または画面全体)を選ぶ
3. あとは会議でページが送られるたびに自動保存。終わったらZIPでまとめてダウンロード

## 仕様

- 自動モード: 「画面が大きく変わった後に静止した」瞬間だけ保存(マウスカーソル程度の動きは無視)。感度3段階
- 映像のノイズ量(フレーム間変化の移動平均)を自動学習し、判定閾値を底上げする適応方式。ノイズで静止が取れない映像でも変化から約2.5秒で必ず保存する取りこぼし防止付き
- 「📌撮影ボタンを最前面に浮かせる」で、常に最前面の小窓に📸ボタンを表示(Chrome/EdgeのドキュメントPiP。非対応ブラウザはポップアップ)。会議画面の隅からワンクリックで撮影できる
- 手動撮影はスペースキーでも可能(メインタブ・浮きボタン小窓の両方で有効。入力欄やセレクトにフォーカス中は誤爆しないよう除外)
- 効果音・通知音は一切なし(保存の合図は緑枠と枚数表示のみ)
- マウスカーソルの写り込み防止: cursor:never指定で映像からカーソルを除去。さらに浮きボタン撮影時は1.2〜2.4秒前の(マウスが近づく前の)コマを保存する二重の対策。ページ送り直後は最新コマに自動フォールバック
- 定間隔モード(N秒ごと・同一画面はスキップ)/ 手動モードも選択可
- プレビューをドラッグすると撮る範囲を指定できる(資料部分だけ切り抜き、検知もその範囲だけになる)
- PNG(文字くっきり)/ JPG(軽い)を選択可、ファイル名は shot_001 からの連番
- ZIP保存で各画像の更新日時=撮影時刻。映像は端末内処理のみで外部送信なし
- 検知は縮小グレースケール画像の「変化ピクセル割合」比較(160px幅・閾値12/255)
